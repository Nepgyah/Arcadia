# Versoning

## Summary
This file contains the current logic and reason behind the version of Arcadia

## Major Versioning (X)
Arcadia increases in major updates (Ex: 2.x -> 3.x) when either:
- A app is finished
- A new major rework/feature is introduced (GraphQL, Redux, etc)

## Minor Versioning (0.X)
Arcadia increases in minor updates (Ex: 2.1.x -> 2.2.x) when:
- Features are added in apps (new pages, ) 

## Patch Versioning (0.0.X)
Patches deal with bug fixes and refactoring

## Micro Versioning (0.0.0.X)
Micro or hotfixing occurs on Arcadia for changes that need intermediate (i.e work on release)

## Release tags
- Arcadia will convert from `alpha` to `beta` when **All primary apps reach their version 1.0 goals**
- Arcadia will conver from `beta` to `release` when **All secondary apps reach their version 1.0 goals**
